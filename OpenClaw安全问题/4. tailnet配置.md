# 如果想让自己的同一个tailnet都可以访问UI界面：

## 1. 启动服务

```
openclaw gateway --tailscale serve
```



## 2. 访问配置以及批准设备

```
openclaw config set gateway.bind loopback
openclaw config set gateway.tailscale.mode serve
openclaw config set gateway.auth.mode token
openclaw config set gateway.auth.allowTailscale false
openclaw config set gateway.trustedProxies '["127.0.0.1","::1"]'


# 这里要改成运行自己UI服务的那个VPS所对应的magicdns
openclaw config set gateway.controlUi.allowedOrigins '["https://vps-2a3a4c57.tail298f6e.ts.net"]'
```

```
openclaw devices list
openclaw devices approve <requestId>
openclaw devices reject <requestId>
```



## 3. 如何访问

```
https://<magicdns>/   -->   Tailscale Serve HTTPS 入口   -->   127.0.0.1:18789
```

```
同一 tailnet 的另一台设备
        |
        |  访问 https://<magicdns>/
        v
Tailscale MagicDNS 解析到你的 VPS
        |
        v
Tailscale Serve 的 HTTPS 入口
        |
        |  反向代理
        v
127.0.0.1:18789 上的 OpenClaw Gateway
        |
        v
Control UI / WebSocket / API
```
