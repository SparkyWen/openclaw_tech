# 第 0 步：先更新系统并装基础工具

Ubuntu 的安全更新默认通过 `unattended-upgrades` 自动处理，默认每天运行一次；但在你开始部署前，先把系统更新到最新仍然是必要的。

```
apt update && apt full-upgrade -y
apt install -y curl git vim ufw fail2ban unattended-upgrades ca-certificates
reboot
```

重连后继续。

---



# 第 1 步：先建一个普通管理员用户，并给它设置 sudo 密码

## 如果是直接root显示登陆的需要做！

在 Ubuntu 里，`sudo` 默认让你输入的是**当前用户自己的密码**。
 所以正确做法不是先给 root 搞密码，而是：

- 创建一个普通用户
- 把它加入 `sudo` 组
- 给这个用户设密码
- 以后用这个用户 + `sudo`

## 推荐这样做

下面我用 `ubuntu` 举例，你可以换成你想要的用户名。

```
adduser ubuntu
usermod -aG sudo ubuntu
passwd
```

然后把你当前 root 账户里的 SSH 公钥复制给这个用户：

```
rsync -a --chown=ubuntu:ubuntu /root/.ssh /home/ubuntu/
chmod 700 /home/ubuntu/.ssh
chmod 600 /home/ubuntu/.ssh/authorized_keys
```

现在**不要退出当前 root 会话**，新开一个终端测试：

```
ssh ubuntu@你的公网IP
```

---



# 第 2 步：立刻做 SSH 加固

OpenSSH 文档里，`PasswordAuthentication` 默认是 `yes`，`PubkeyAuthentication` 默认是 `yes`，SSH 端口默认是 `22`。你既然已经用 SSH key 登录了，就应当把密码登录关掉，并最终禁止 root 远程登录。

```
/etc/ssh/sshd_config
```

```
PermitRootLogin no
PasswordAuthentication no
KbdInteractiveAuthentication no
PubkeyAuthentication yes
UsePAM yes
```

然后检查配置有没有语法错误：

```
sshd -t
```

如果没报错，再重载 SSH：

```
systemctl reload ssh
```

## 这一步一定这样验证

1. **当前 root 会话不要关**
2. 新开一个窗口
3. 用 `ubuntu` 再 SSH 一次测试能否登录
4. 测试成功后，才算这一步完成

# 第 3 步：装 Tailscale，并先验证 Tailscale SSH 路径

Tailscale 官方 Linux 安装文档给出的主流发行版安装方式是：

```
curl -fsSL https://tailscale.com/install.sh | sh
sudo tailscale up
```

完成后会给你一个浏览器授权链接；授权成功后，机器会出现在你的 tailnet 里。官方也建议服务器场景考虑关闭 key expiry，避免以后远程被锁在门外。

## 安装与接入

```
curl -fsSL https://tailscale.com/install.sh | sh
tailscale up
```

然后查看这台 VPS 的 Tailscale IP：

```
tailscale ip -4
```

你会得到一个 `100.x.y.z` 的地址。

## 从你本地另一台已经登录 Tailscale 的设备测试

```
ssh david@100.x.y.z
```

**先确认这条路能通**，再去收公网 SSH。

## 很关键：关掉这台服务器的 Tailscale key expiry

Tailscale 文档说明可以在管理后台 Machines 页面把某台设备的 key expiry 关闭；如果 key 过期，连接会停止工作。对于服务器，关掉 expiry 通常更稳。

另外，官方也提醒：如果设备已经过期，不要只靠远程 Tailscale SSH 去做 `tailscale up --force-reauth`，因为这可能把你当前远程连接带断。

# 4. UFW防火墙

## 我建议你直接执行的命令

先确认接口名没看错：

```
ip -br a
```

确认公网网卡确实还是 `ens3`，Tailscale 还是 `tailscale0`。
 然后按下面执行。

## 你现在这台机的网卡名不是 `ens3`

你当前看到的是：

```
lo
eth0
eth1
```

所以你之前那套：

```
22/tcp on tailscale0
22/tcp on ens3
```

在这台新 VPS 上**不能直接照搬**，因为：

- **没有 `ens3`**
- **目前也还没有 `tailscale0`**

------

## 先把你当前网络状态解释清楚

你现在的输出是：

```
lo               UNKNOWN        127.0.0.1/8 ::1/128 
eth0             UP             170.64.176.61/20 10.49.0.5/16 fe80::...
eth1             UP             10.126.0.2/20 fe80::...
```

## 这代表什么

### `lo`

本机回环接口，不用管。

### `eth0`

这是你当前最关键的主网卡。
 因为它带了公网 IPv4：

```
170.64.176.61
```

所以你现在公网访问这台机器，主要就是走 **`eth0`**。

### `eth1`

这是另一块私网接口，只有私网地址：

```
10.126.0.2
```

通常用于私有网络、VPC、内网通信之类。

------

# 现在最重要的两个结论

## 结论 1：你的公网 SSH 封禁规则应该针对 `eth0`

不是 `ens3`。

也就是说，你如果要实现：

> “公网不允许 SSH，只允许 Tailscale SSH”

那你应该写的是：

```
sudo ufw deny in on eth0 to any port 22 proto tcp
```

不是：

```
sudo ufw deny in on ens3 to any port 22 proto tcp
```

------

## 结论 2：你现在还没有 Tailscale

因为如果 Tailscale 已经正常连上，`ip -br a` 里通常会出现：

```
tailscale0
```

但你现在没有。
 这说明：

- 要么你**还没安装 Tailscale**
- 要么你**装了但还没 `tailscale up`**
- 要么它**还没成功连上**

所以你现在**不能直接认为 SSH 已经能走 Tailscale**。

------

# 你现在正确的操作顺序

你这台机器现在还是 root 登录状态，所以我给你一个**不会把自己锁死**的顺序。

------

# 第一步：先不要立刻封死公网 22

因为你现在还没有 `tailscale0`。
 如果你这时候直接：

```
ufw deny in on eth0 to any port 22 proto tcp
```

而 Tailscale 还没连好，你就可能把自己远程锁在外面。

所以现在正确做法是：

## 先确认当前 SSH 入口还保留

先看现在防火墙状态：

```
ufw status verbose
```

如果还没启用，先别急着上最严规则。

------

# 第二步：先把 Tailscale 装好

你现在先执行：

```
curl -fsSL https://tailscale.com/install.sh | sh
tailscale up
```

执行 `tailscale up` 后，它会给你一个链接，让你在浏览器里授权登录。

完成后再执行：

```
tailscale ip -4
ip -br a
```

### 到时候你应该能看到

`ip -br a` 多出一行类似：

```
tailscale0       UNKNOWN        100.x.y.z/32
```

这时候才说明 Tailscale 真正起来了。

------

# 第三步：先测试 Tailscale SSH 能不能通

拿到 Tailscale IP 后，比如：

```
100.x.y.z
```

你在你自己电脑上测试：

```
ssh root@100.x.y.z
```

或者如果你后面切普通用户，就：

```
ssh ubuntu@100.x.y.z
```

## 只有这一步成功之后

你才可以安全地封公网 SSH。

------

# 第四步：按你这台机的真实网卡名写 UFW

如果你的目标仍然是你之前那套思路：

- 80/443 对公网开放
- 22 只允许 Tailscale
- 公网网卡禁止 22

那么在**这台机器**上应该写成：

```
ufw default deny incoming
ufw default allow outgoing

ufw allow 80/tcp
ufw allow 443/tcp
ufw allow in on tailscale0 to any port 22 proto tcp
ufw deny in on eth0 to any port 22 proto tcp
```

注意：

- 这里是 `eth0`
- 不是 `ens3`

# F. 配 Fail2ban

```
cat >/etc/fail2ban/jail.d/sshd.local <<'EOF'
[DEFAULT]
bantime = 1h
findtime = 10m
maxretry = 5
backend = systemd
banaction = ufw
ignoreip = 127.0.0.1/8 ::1 100.64.0.0/10

[sshd]
enabled = true
EOF

systemctl enable --now fail2ban
systemctl restart fail2ban
fail2ban-client status sshd
```

这里的核心意思是：

- 本机回环不封
- Tailscale CGNAT 网段 `100.64.0.0/10` 不封



## 你这段配置本身为什么能用

这几项都是 Fail2ban jail 的标准参数：

- `bantime`：封禁时长
- `findtime`：统计失败次数的时间窗
- `maxretry`：达到多少次失败后触发封禁
- `backend`：日志后端；Ubuntu 的手册里说明可用 `systemd`
- `banaction`：封禁动作；可以在 `[DEFAULT]` 里统一指定
   这些都在 Ubuntu 的 Fail2ban `jail.conf` 手册里有定义。







## 如果你是在自己的 Win / Mac 本机运行 OpenClaw

**不是“只需要装 Tailscale”**，而是：

1. **本机只用 localhost / 127.0.0.1 跑 OpenClaw** 不要暴露公网
2. **保持 OpenClaw 默认的 token auth**
3. **不要关闭 device auth / pairing**
4. **需要跨设备远程访问时，再装 Tailscale**

也就是说：

> **Tailscale 不是本地单机运行的必需品，而是“你想从别的设备安全访问这台机器上的 OpenClaw”时特别推荐的方案。**
>  本机自己用，哪怕不装 Tailscale，也可以是安全的。
